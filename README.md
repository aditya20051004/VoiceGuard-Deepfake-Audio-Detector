**🎙️ Deepfake Audio Detector**

An end-to-end AI-powered Deepfake Audio Detection system built using PyTorch, CNNs, and Mel Spectrogram analysis to classify whether an audio clip is REAL or AI-generated.

This project focuses on detecting synthetic speech generated by modern TTS (Text-to-Speech) engines such as:

* Google gTTS
* Microsoft Edge-TTS
* eSpeak
* Other AI-generated synthetic voices

The model is trained on transformed audio spectrograms and deployed with an interactive Gradio UI for real-time testing.

⸻

**🚀 Features**

✅ Deepfake audio classification using CNNs

**🧠 Model Architecture**

The system uses a custom-built Convolutional Neural Network (CNN) trained on Mel Spectrogram representations of audio.

Architecture Highlights

* Input Shape: 1 x 128 x 128
* Multiple Conv2D + BatchNorm blocks
* ReLU activations
* MaxPooling layers
* Dropout regularization
* Fully connected classifier head
* Binary classification output (REAL vs FAKE)

📊 Pipeline Overview

1️⃣ Audio Collection

Real Audio

* LJSpeech Dataset
* Human speech recordings

Fake Audio

Generated using:

* gTTS
* Edge-TTS
* eSpeak

⸻

2️⃣ Audio Preprocessing

Each audio file undergoes:

* Resampling → 16kHz
* Duration normalization → 4 seconds
* Mono conversion
* Padding / trimming
* Mel Spectrogram conversion
* dB scaling
* Normalization

⸻

3️⃣ Feature Extraction

Mel Spectrogram parameters:

Parameter	Value
Sample Rate	16000
Duration	4 sec
Mel Bands	128
FFT Size	2048
Hop Length	512
Target Shape	128 × 128

⸻

4️⃣ Training

Training includes:

* Binary Cross Entropy Loss
* Adam Optimizer
* Gradient Clipping
* Validation Monitoring
* Best Model Checkpoint Saving
* Data Augmentation

Augmentations Used

* Time Masking
* Frequency Masking
* Noise robustness improvements
